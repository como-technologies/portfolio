# Products

Productized offerings in the Como Technologies portfolio. Products are the
durable artifacts and patterns clients adopt — the output of the
[Prescribe](../loop/prescribe.md) stage.

The portfolio has one product, deliberately: **the playbook-as-artifact
pattern is the product**, and it ships as a working repository, not a
document format.

- **[The playbook](./playbook.md)** — a generic, ready-to-adopt playbook
  template: an adroit-managed ADR corpus plus guides, published as a
  self-hosted mdBook, CI-gated, and readable by the Adopt stage's engine
  over a machine seam. *Maturity: dogfooding* — it is the corpus the
  portfolio's own loop exercises every iteration.
- **palette-playbook** — the delivered instance of the same pattern, built
  for an enterprise cloud and Kubernetes platform team. It is described as
  the product page's [proof point](./playbook.md#the-delivered-proof-point).

Each engagement produces a tailored variant — opinionated where the client
needs a jumpstart, flexible where they have their own conventions. The
[Playbook authoring](../services/playbook-authoring.md) service is how a variant gets
made; the [Agentic delivery](../services/agentic-delivery.md) pilot is how
its decisions get worked.
